# Aligator Rent CRM Mini App v0.3.3 — Bike Health / пробег / ТО

База: v0.3.1 contract fields.

## Что добавлено

### DB
- `bike_odometer_reports` — история пробега по каждому велику.
- `bike_service_events` — ручные сервисы/ремонты/замены/ТО.
- `bike_maintenance_tasks` — задачи ТО, когда после сервиса проехали около 1000 км.
- `miniapp_notifications` — центр уведомлений, чтобы не засорять Telegram.
- `miniapp_bike_health_summary` — карточка состояния велика.
- `miniapp_bike_battery_health` — привязанные батареи и их возраст/состояние.
- безопасные поля в `batteries`: `first_used_at`, `health_status`, `health_notes`, `last_checked_at`.

### Mini App
- новая вкладка админа `🧰 Состояние`;
- карточка вида: текущий пробег, последний сервис, после сервиса, до ТО, статус;
- ручной ввод пробега по одному велику;
- массовая форма пробега: `24: 1840`;
- ручное создание сервисного события/ремонта;
- отображение последних ремонтов;
- отображение батарей, привязанных к велику;
- клиентский кабинет показывает состояние active велика и форму отправки пробега.

## Важно по SQL

Файл `sql/001_bike_centered_miniapp.sql` теперь в начале удаляет только Mini App views через `DROP VIEW ... CASCADE`, чтобы PostgreSQL не падал на ошибках смены порядка колонок. Реальные таблицы и данные не удаляются.

Запустить в Supabase SQL Editor нужно весь файл.

## Тест после деплоя

1. Admin Mini App → `🧰 Состояние`.
2. В массовой форме ввести:
   ```
   24: 1840
   ```
3. Открыть карточку #24.
4. Записать сервис `Простое ТО` на текущем пробеге.
5. Потом ввести пробег +1000 км.
6. Проверить, что появилась задача ТО.
7. Зайти как клиент и отправить пробег через клиентский кабинет.

## Roapp.io

Roapp.io пока не подключён. Новая структура уже подготовлена: будущий импорт ремонтов должен писать в `bike_service_events`.

## v0.3.3 update: батареи

В карточке велика в блоке `Батареи` можно указать ID батарей, например `1, 2`, и привязать их к выбранному велику. Пока это обновляет `batteries.bike_id`, `status`, `first_used_at`, `health_status`, `health_notes`, `last_checked_at`. Более точная история замен батарей через отдельные события уже будет следующим слоем.
